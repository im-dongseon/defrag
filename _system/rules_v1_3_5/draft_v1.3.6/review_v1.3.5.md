# Review for defrag Rules v1.3.5

**일자:** 2026-04-09
**대상 버전:** v1.3.4 -> v1.3.5 (Patch)
**주요 목표:** 도구 활용 계층화 및 OS 명령어 보안/효율성 강화 규칙 전역 반영

## 1. 요구사항 분석 및 반영 계획

### [REQ-1] 도구 활용 계층화 (1/2/3 순위)
- **현황:** `index.md` 및 `R2_search.md`에 1/2순위만 명시되어 있거나, 에이전트 도구(`grep_search`)가 2순위로 지정되어 토큰 낭비 위험이 있음.
- **반영 계획:** 
  1. **1순위:** Obsidian CLI (인덱스/메타데이터 정밀 작업)
  2. **2순위:** OS 표준 명령어 (`mv`, `\grep`, `\find`, `\cat` 등 - Alias 우회 필수)
  3. **3순위:** 에이전트 전역 도구 (`read_file`, `replace`, `grep_search` 등 - 정밀 수정 및 대형 파일용)
- **영향 파일:** `index.md`, `R2_search.md`

### [REQ-2] Alias Bypass 및 데이터 무결성
- **현황:** 사용자가 `cat`을 `bat` 등으로 alias한 경우 라인 넘버/색상 코드가 삽입되어 에이전트의 데이터 처리가 실패함.
- **반영 계획:** `run_shell_command`를 통한 텍스트 조회 시 반드시 `\` (예: `\cat`, `\grep`)를 붙여 Alias를 우회하도록 명시.
- **영향 파일:** `index.md`, `R2_search.md`

### [REQ-3] 파일 크기 임계치 준수 (100줄/10KB)
- **현황:** 대형 파일에 `cat`을 사용할 경우 응답 절단(Truncation)으로 인한 데이터 유실 위험.
- **반영 계획:** 100줄 또는 10KB 이상의 파일은 반드시 에이전트 도구(`read_file`)의 `start_line/end_line` 기능을 사용하도록 강제.
- **영향 파일:** `index.md`, `R2_search.md`

### [REQ-4] 파일 이동 후 볼트 인덱스 갱신 (Reload)
- **현황:** `mv`로 파일 이동 시 Obsidian 내부 인덱스가 즉시 갱신되지 않아 에이전트가 이동된 파일을 찾지 못하는 현상 발생.
- **반영 계획:** 파일 이동(`mv`) 후 반드시 `obsidian eval` 명령어를 사용하여 인덱스 갱신을 유도하도록 절차 추가.
- **영향 파일:** `R1_ingest.md`, `R4_lint.md`
