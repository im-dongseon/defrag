## v1.3 계획 과제

### 1. Dashboard 역할 이원화 분리
시스템 캐시(`[Cache]`, `[Lint]`)와 사용자 작업 로그(`[Log]`)를 물리적으로 분리합니다.
- 검토 방향: `cache.md` 별도 파일 도입
- 영향 파일: `R4_lint.md`, `R8_security.md`

### 2. `82_index.md` sources 요약 통합
`82_Synthesis/` 인덱스 파일에 `sources` 요약 정보를 포함하여, 전체 파일 스캔 없이 인덱스만으로 부모 노드 파악이 가능하도록 개선합니다.

### 3. 전체 룰 토큰 비용 재점검
v1.1.0 컴포넌트 분리 이후 실제 운영 부하를 측정하여 추가 최적화 포인트를 발굴합니다.

### 4. 유사도 판단 임계값 정책 단일화
`R1_ingest.md`와 `R5_synthesis.md`에 분산 선언된 80% 유사도 기준을 단일 정책으로 승격하고, 개별 규칙은 참조만 유지하도록 정리합니다.

### 5. unsorted/ 반복 실패 에스컬레이션 정책 추가
도메인 판단이 지속 실패하는 파일은 매 Lint 사이클마다 `[Log] unsorted 재배치 불가` 로그를 반복 기록하게 됨. N회 실패 후 사용자에게 명시적으로 보고하는 에스컬레이션 정책을 `R4_lint.md` Step 5에 추가합니다.
- 검토 방향: 실패 횟수를 Dashboard 로그에서 집계, 임계값(예: 3회) 초과 시 별도 `[Log]` 포맷으로 사용자 보고

### 6. 추출 실패 Sidecar 파일명 규격 명시
`R1_ingest.md` 1단계 추출 실패 시 생성하는 임시 Sidecar의 파일명 형식이 정의되어 있지 않음. 충돌 방지를 위해 규격을 명시합니다.
- 검토 방향: `[sidecar] 원본파일명.md` 또는 `[YYYYMMDDHHmm] 원본파일명_sidecar.md` 등 후보 검토

### 7. unsorted/ 반복 처리 방지 메커니즘
Lint 사이클 간 재배치 실패 파일과 신규 유입 파일을 구분하는 장치가 없어 매 사이클마다 동일 파일을 재시도함. 불필요한 처리를 줄이는 방안을 검토합니다.
- 검토 방향: Dashboard `[Unsorted]` 로그의 날짜 기반으로 최초 기록 이후 경과 사이클 추적, 또는 별도 `skip` 플래그 도입
