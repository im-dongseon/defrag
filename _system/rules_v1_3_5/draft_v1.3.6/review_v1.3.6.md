## 1. 요구사항 분석 및 반영 계획

### [REQ-1] 파일명 중복 배제 (Deduplication)
- **현황:** 에이전트가 `30_Project-X_회의록_2026-04-09.md`와 같이 상위 폴더명을 파일명에 중복 기재하여 경로가 불필요하게 길어짐.
- **반영 계획:** 파일명에 상위 폴더명(카테고리명, 프로젝트명) 기재를 금지하고 `[번호] 주제_날짜.md` 형식을 강제함.

### [REQ-2] 동일 날짜 다중 파일 대응
- **현황:** 하루에 여러 회의가 있을 경우 파일명 충돌 위험.
- **반영 계획:** `_n` (차수) 또는 `_세부주제` 접미사 허용 규칙 추가.

### [REQ-3] 바이너리/미디어 처리 프로세스 강화 (Sidecar 우선순위)
- **현황:** 오디오(.m4a) 등 일부 바이너리 파일 처리 시, 텍스트 추출 실패를 이유로 Sidecar 생성을 생략하고 원본만 아카이브로 이동시키는 결함 발견.
- **반영 계획:** 
  - 바이너리/미디어 확장자(오디오, 비디오, 이미지, PDF 등)를 명확히 정의.
  - 어떠한 경우에도 **Sidecar(.md) 파일이 생성되기 전에는 원본 파일을 이동할 수 없음**을 명시.
  - 추출 실패 시에도 `status: idea`의 빈 Sidecar를 즉시 생성하여 원본과의 연결성을 보장.

### [REQ-4] 아카이브 경로 정규화 및 Lint 강화
- **현황:** 아카이브(`91_Raw_Archive/assets/`) 내 도메인 폴더명이 위키의 메인 폴더명(`[번호]_[이름]`)과 일치하지 않는 사례(예: `Project-X` vs `30_Project-X`) 발생.
- **반영 계획:**
  - 아카이브 내 도메인 폴더명은 반드시 위키의 메인 폴더명과 100% 일치하도록 강제.
  - `R4 Lint` 단계에서 아카이브 폴더명의 정합성을 검사하는 로직 추가.
