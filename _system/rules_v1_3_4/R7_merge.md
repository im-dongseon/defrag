## 중복 징후 감지 및 보고

R4 Lint 주기 중 기존 위키 노트들 사이를 관측합니다.  
아래 조건을 만족하는 **징후쌍(병합 후보군)이 탐지된 경우**, 즉시 통합하지 않고 Dashboard `[Log]`에 병합 후보로 보고합니다.

| 조건 | 기준 |
|------|------|
| 태그 중복 | 동일 태그 4개 이상 동시 공유 |
| 헤딩 중복 | 동일 최상위 헤딩(`# 제목`) 3개 이상 일치 |

---

## Rewrite 통합 절차 (80% 유사 판정 시)

아래 순서로 기존 노트에 신규 내용을 통합합니다.

1. **중복 단락 처리:** 신규 내용 중 기존 노트와 완전히 동일/유사한 단락은 폐기합니다.  
   단, 신규 정보가 **더 최신이거나 구체적**이면 기존 낡은 단락을 신규 정보로 **덮어쓰기(대체)** 합니다.
2. **신규 정보 삽입:** 신규에만 존재하는 정보를 기존 노트의 관련 섹션에 추가합니다.
3. **메타데이터 갱신:** 정리 완료 직후 `updated` 필드를 현재 KST 시각으로 갱신합니다.

병합 후 잔여 낡은 정보는 `99_History/`에 보존합니다.  
**단, `82_Synthesis/`의 `type: synthesis` 노트는 이 분리 이동 대상에서 영구 제외합니다.**

## 하드 분리 임계치

변동 폭이 **300자 이상이며 전체 볼륨의 30% 이상**이 교체될 때, 과거 원본을 이관합니다.

- **이관 경로:** `99_History/[파일명]_history.md`
- **이관 인가:** R8의 Archive 쓰기 예외 조항에 의해 허용됩니다.

### `_history.md` 파일 헤더 규격

YAML Frontmatter를 절대 사용하지 않으며, 맨 윗 2줄 텍스트만 기록합니다.

```
# [원본 파일명] History
이관 사유: [사유] | 이관 일자: [YYYYMMDD]
```
