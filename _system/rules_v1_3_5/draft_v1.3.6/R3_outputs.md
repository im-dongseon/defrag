# R3. 질의 응답 결과물 보존 (Outputs Feedback Loop) 및 Quality Gate

## 저장 원칙

Quality Gate 조건 충족 시에만 `81_Outputs`에 저장하며 Dashboard에 `[Log]`로 기재합니다.

## Quality Gate (저장 품질 기준)

Output 공간의 무의미한 비대화를 저지합니다. 아래 3가지 중 하나에 해당하는 **정보 밀도가 높은 결과물**에 한정하여 저장합니다.

| 저장 대상 | 설명 |
|-----------|------|
| ✅ 구조화 응답 | 분석표·비교표·구조화된 요약문 포함 응답 |
| ✅ 설명형 응답 | 3문장(약 150자) 이상의 설명형 응답 |
| ✅ 명시 지시 | 사용자가 명시적으로 저장을 지시한 출력 |
| ❌ 일회성 로그 | "확인했습니다", "완료했습니다" 등 단발 확인 응답 |

## 보안 및 PII 저장 제한

답변 결과물에 PII(개인식별정보)가 발견될 경우 **저장 절차는 전면 보류**되며, Dashboard에 `[Log] PII 및 보안 정보 감지` 기재로 갈음합니다.  
PII 마스킹(R8)은 화면 출력 및 로그 기재에만 적용되며, 파일 저장에는 이 규칙(R3)이 우선합니다.

## 출력물 규격

- **파일명:** `[분류번호] YYYYMMDD_간단한_주제.md`
- **Frontmatter:** `type: output`
