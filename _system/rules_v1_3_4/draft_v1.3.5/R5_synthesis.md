## 합성 중복 방지 절차

### 0. 인덱스 부트스트랩 (전제 조건)

`82_index.md`가 없거나 파일이 손상된 경우, 에이전트는 빈 인덱스를 생성하거나 기존 `82_Synthesis/` 파일을 직접 스캔하여 `82_index.md`를 재구성한 뒤 합성 절차를 계속합니다.

### 1. 중복 사전 확인

신규 작성 전, **`82_index.md`를 먼저 로드**하여 제목과 태그를 최소 1차 검사합니다.

- **80% 이상 유사한 기존 자료 발견 시:** 새 생성 중단 → 기존 내용의 갱신(Rewrite)으로 대체 대응
- **유사 자료 없음:** 신규 노트 생성 진행

### 2. 인덱스 동기화 의무

신규 노트 생성 직후 → **에이전트가 직접 즉시** `82_index.md`에 새 문서의 제목과 핵심 태그를 업데이트합니다.

## 합성 산출물 정기 점검 (R4 Lint 7단계 연동)

R4의 1~6단계에서 수정·보강된 노트들의 파일 경로 리스트를 취합하여, **`82_Synthesis/` 내 각 파일의 `sources` 필드를 역조회(역참조)** 합니다.

### 역조회 방법

| Obsidian 상태 | 방법 |
|--------------|------|
| **활성 시** | `obsidian eval`로 `sources` 필드 일괄 조회 |
| **미활성 시** | `82_Synthesis/` 디렉토리 직접 스캔 + `sources` 필드 전역 텍스트 검색 (index.md 도구 대원칙 2순위 참조) |

### 점검 결과 처리

- `sources` 필드에 수정된 노트 경로가 포함된 합성 노트 → **내용 유효성 심사 강제 편입**
- 내용이 현저히 낡음 → `status: deprecated`로 변경
- 필요 시 새 합성을 모색
